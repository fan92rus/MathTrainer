## 1. Executive Summary

### 1.1 Vision

Заменить город-строительство (Phaser) как основную мотивационную систему на **лёгкую многослойную мотивацию Duolingo-стиля**: ежедневные стрики, визуальный прогресс-путь по темам и улучшенные ачивки. Это обеспечит устойчивое вовлечение без дорогостоящей 3D-игры.

### 1.2 Value Matrix

| Для разработчиков | Для пользователей (дети 6–12) |
|---|---|
| Удаление Phaser-зависимости (~50 КБ bundle) | Понятная визуальная мотивация |
| Простое добавление новых механик | Ежедневная привычка через стрики |
| Меньше кода — меньше багов | Чувство прогресса («вижу свой путь») |
| Не нужен отдельный «игровой» контекст | Мгновенная награда за усилия |

### 1.3 Научное обоснование

- **Стрики** → автономия + компетентность (SDT, Li et al. 2024, g = 0,638)
- **Прогресс-путь** → автономия (вижу куда иду) + компетентность
- **Ачивки** → компетентность (мастерство) без токсичного соревнования
- **Многослойность** → борьба с эффектом новизны (Rodrigues et al. 2022: U-образная кривая)

## 2. Problem Statement

### 2.1 Текущее состояние

| Проблема | Влияние |
|---|---|
| Город на Phaser — тяжёлая зависимость, сложная в разработке | Высокая стоимость изменений |
| Мотивация исключительно внешняя («реши → получи → построй») | «Шоколадная брокколи» — дети играют ради города, не ради математики |
| Нет системы ежедневного возврата | Низкая Retention Day 7+ |
| Ачивки есть, но disconnected от основного loop | Воспринимаются как «доп. экран», а не часть опыта |
| Нет визуального пути обучения | Ученик не видит, сколько осталось и что дальше |

### 2.2 Что уходим

- ❌ Город (Phaser) — убираем из навигации и роутинга
- ❌ Монеты/кристаллы/здания — выводим из основного loop
- ❌ Валюта как основная мотивация

### 2.3 Что сохраняем

- ✅ Существующие ачивки (данные и типы) — мигрируем
- ✅ Ежедневные задания — интегрируем в новую систему
- ✅ Player store — рефакторим под новые нужды

## 7. Functional Requirements

### 7.1 FR-1: Ежедневные стрики (Streaks)

#### 7.1.1 Определение «дня активности»

«Активный день» = решён хотя бы **1 пример** в любом упражнении. Проверка по локальной дате `new Date().toDateString()`.

#### 7.1.2 Логика стрика

```
При каждом решении примера:
  1. Получить lastActiveDate из store
  2. today = new Date().toDateString()
  3. Если today === lastActiveDate → ничего не делаем (уже активен сегодня)
  4. Если today === yesterday(lastActiveDate) → streak++, обновить lastActiveDate
  5. Если lastActiveDate старше yesterday → streak = 1, обновить lastActiveDate
```

#### 7.1.3 Отображение стрика

| Элемент | Описание | Где |
|---|---|---|
| 🔥 Огонь стрика | Анимация пламени, размер зависит от streak | MotivationBar (верх экрана) |
| Счётчик | Число дней рядом с огнём | MotivationBar |
| Максимум | Лучший стрик (сохраняется) | Профиль/настройки |

#### 7.1.4 Вехи стрика (стимулы)

| Стрик | Награда | Анимация |
|---|---|---|
| 3 дня | Бейдж «Начало пути» | Маленький огонёк |
| 7 дней | Бейдж «Неделя практики» | Средний огонь + конфетти |
| 14 дней | Бейдж «Две недели!» | Большой огонь |
| 30 дней | Бейдж «Месяц мастерства» | Радужный огонь + анимация |

#### 7.1.5 «Заморозка» стрика (опционально, v2)

Позволить пропустить 1 день без сброса стрика. Лимит: 1 заморозка в неделю. Реализация — отложена до v2.

### 7.2 FR-2: Прогресс-путь (Learning Path)

#### 7.2.1 Структура пути

Визуальный «путь» (вертикальный или горизонтальный) с узлами по темам:

```
📁 Класс 1
  └─ 📁 Четверть 1
       ├─ ⬜ Счёт до 10        → nodes: 5 (пороги: 0/20/40/60/100 очков)
       ├─ ⬜ Счёт до 20        → nodes: 5
       └─ ⬜ Разложение        → nodes: 5
  └─ 📁 Четверть 2
       ├─ ⬜ Уравнения          → nodes: 5
       └─ ...
📁 Класс 2
  └─ ...
```

#### 7.2.2 Узел пути (PathNode)

```typescript
interface PathNode {
  id: string
  exerciseType: string       // 'counting' | 'multiplication' | ...
  title: string              // 'Счёт до 10'
  description: string        // 'Сложение и вычитание в пределах 10'
  thresholds: number[]       // [0, 20, 40, 60, 100] — очки для звезды
  starCount: 0 | 1 | 2 | 3  // вычисляется по текущему score
  locked: boolean            // зависит от класса/четверти
  icon: string               // emoji или SVG
}
```

#### 7.2.3 Вычисление звёзд

```typescript
function getStars(score: number, thresholds: number[]): 0 | 1 | 2 | 3 {
  if (score >= thresholds[3]) return 3  // максимальный порог
  if (score >= thresholds[2]) return 2
  if (score >= thresholds[1]) return 1
  return 0
}
```

#### 7.2.4 Отображение

- Горизонтальный скролл на HomeView (основной экран)
- Каждый узел — карточка с иконкой, названием и звёздами
- Заблокированные узлы — серые с 🔒
- Текущий узел — подсвечен, пульсирующая рамка
- Пройденные — зелёная галочка + звёзды

### 7.3 FR-3: Улучшенные ачивки

#### 7.3.1 Toast-уведомления

При разблокировке ачивки показывается **toast** в верхней части экрана (не модалка):

```
┌─────────────────────────────────┐
│ 🏆 Разблокировано:             │
│ 🔥 Идеальная серия             │
│ 50 правильных ответов подряд    │
│           [Отлично!]            │
└─────────────────────────────────┘
```

- Появляется поверх контента
- Автоматически исчезает через 4 сек
- Можно закрыть свайпом/кнопкой
- Максимум 1 toast за раз (очередь)

#### 7.3.2 Новые категории ачивок

Добавить категорию `streak_daily` в `Achievement.category`:

```typescript
// Новые ачивки для ежедневных стриков
{ id: 'streak_3', name: 'Начало пути', description: '3 дня подряд', category: 'streak_daily', icon: '🔥', ... }
{ id: 'streak_7', name: 'Неделя практики', description: '7 дней подряд', category: 'streak_daily', icon: '🔥', ... }
{ id: 'streak_14', name: 'Две недели!', description: '14 дней подряд', category: 'streak_daily', icon: '🔥', ... }
{ id: 'streak_30', name: 'Месяц мастерства', description: '30 дней подряд', category: 'streak_daily', icon: '🔥', ... }
```

#### 7.3.3 Интеграция с основным loop

Сейчас `checkAchievements()` вызывается после упражнения. Дополнительно:

- Вызвать `checkStreakAchievements()` при обновлении стрика

## 10. Testing Strategy

### 10.1 Unit-тесты

| Что тестируем | Кейс | Ожидание |
|---|---|---|
| Streak store | Первый вход | streak = 1, lastActiveDate = today |
| Streak store | Вход на следующий день | streak = 2 |

- Решить пример → проверить обновление стрика
- Решить пример → проверить обновление узла пути
- Набрать enough очков → проверить разблокировку ачивки + toast

### 10.3 Визуальные тесты

- MotivationBar на разных размерах экрана
- ProgressPath скролл на HomeView
- Toast анимация появления/исчезновения

## 11. Risks & Mitigations

| Риск | Вероятность | Влияние | Митигация |
|---|---|---|---|
| Дети расстроятся из-за удаления города | Средняя | Средняя | Показать «Скоро будет новое!» + стрики как замена |
| Стрики могут демотивировать при потере | Средняя | Высокое | В v2 добавить «заморозку» (1 пропущенный день без сброса) |
| localStorage переполнится | Низкая | Низкое | Данные стриков = <1 КБ, периодическая очистка |
| Часовые пояса влияют на определение «дня» | Низкая | Среднее | Использовать `toDateString()` — всегда локальное время |
